Hybrid Token-Efficient Routing Agent

AMD Hackathon ACT II · Track 1

An intelligent Docker-based AI pipeline that classifies tasks and routes them
to the cheapest capable Fireworks AI model — in zero extra tokens.

Fireworks AI Python Docker OpenAI SDK Regex

The Problem

🔥 Token Waste

Every task sent to the wrong model wastes tokens. A sentiment analysis on a code-generation model can burn 5x the necessary tokens.

🎯 LLM Classification Overhead

Using a second LLM call just to classify the task adds 50-200 tokens of overhead — before even solving the problem.

📊 Model Proliferation

5 allowed Fireworks models across 8 categories = 40 combinations. Picking the right one manually is impossible at scale.

⚡ Latency Sensitivity

Each multi-turn classification call adds 2-5 seconds. In production pipelines, this compounds quickly.

Pipeline Architecture

📥 Input Task
→
🔍 Regex Classifier
→
🧠 Empirical Router
→
🚀 Fireworks Model
→
📤 JSON Output
  • Zero-Token Decision: The regex classifier reads the prompt text pattern and categorizes it — no API call, no token cost, < 1ms latency.
  • Empirical Routing: A routing table built from real benchmark runs across all 5 allowed models picks the cheapest model per category.
  • Category-Specific Optimization: Each category gets a tuned system prompt, response_format constraints, and max_tokens caps.
  • Fallback Strategy: If the response is empty or malformed, the agent retries with the most reliable model.

Token Optimization Strategy

0

Classification Tokens

Regex-based classifier runs locally with zero API cost. Saves 50-200 tokens per task vs LLM-based routing.

JSON

Response Format

JSON mode for sentiment + NER prevents thinking/reasoning tokens from leaking into output. response_format={"type":"json_object"}

max_tokens

Tight Token Caps

Category-specific max_tokens: 200 for sentiment, 3000 for logic. Prevents runaway generation. temperature=0.0
